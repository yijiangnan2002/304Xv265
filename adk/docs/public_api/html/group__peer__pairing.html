Detailed Description

Bluetooth domain functionality for creating paired peer devices by command.

Module contains API and functionality to add peer pairing records. This is normally done using a Bluetooth link in the peer_pair_le code.

It is recommended that this module is only used from test code.

#include <adk/src/domains/bt/peer_pairing/peer_pairing.h>

Update or replace pairing to a peer with pairing to a new address.

This function is for use to pair two devices without the need of any radio connection. The anticipated use is in a factory setting using pydbg or Device Test Service, or from a charger case.

Note
It does not matter if the devices already have earbud pairing.
The earbuds cannot be connected to an earbud. They should be disconnected first.
Parameters
primaryPointer to the primary device address in the pairing
secondaryPointer to the secondary device address in the pairing
is_primaryWhether this is the device that owns the primary address
randomised_keysNew root keys to be set. This is the encryption key and identity root. These must be the same on each device in an earbud pair.
bredr_keyPointer to the BREDR link key.
le_keyPointer to the Bluetooth Low Energy long term key.
Returns
TRUE if the pairing was successful and records were created. FALSE if any errors were detected.

#include <adk/src/domains/bt/peer_pairing/peer_pairing.h>

Remove existing peer pairing and request pairing to a specific device.

The anticipated use is in a factory setting using pydbg or the Device Test Service, or from a charger case.

Pairing is performed asynchronously. Use BtDevice_IsPairedWithPeer() to check for completion of pairing. PeerPairLeIsRunning() may be used to see if pairing is in progress.

Note
Existing peer pairing will be removed, but the device should not be active when called. Similar to PeerPairing_AddPeerPairing() the earbuds cannot be connected to an earbud. They should be disconnected first.
Parameters
taskTask that will receive a confirm message when peering has been added PEER_PAIR_LE_PAIR_CFM
targetPointer to the bluetooth device address with which to pair
Returns
FALSE if existing pairing could not be removed. TRUE if any existing pairing has been removed and pairing has been requested.
