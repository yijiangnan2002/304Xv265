◆ device_test_service_mode_t

enum device_test_service_mode_t

Types of mode in which DTS may be configured.

Enumerator
DTS_MODE_DISABLED 

DTS is not enabled.

DTS_MODE_ENABLED 

DTS is enabled and will try to connect on startup. This mode is expected to be used for production testing.

DTS_MODE_ENABLED_IDLE 

Device testing is enabled, but not the device test service. LE connctions will be enabled.

This mode is expected to be used during production to halt application startup for other setup activities, but not for performing production testing.

DTS_MODE_ENABLED_DUT 

Device testing is enabled, but not the device test service.

This mode is expected to be used during production to put the device into RF DUT mode only.

◆ dts_casecomms_mid_t

enum dts_casecomms_mid_t

Type of DTS message ID used over case comms.

The DTS case comms channel has two sub channels, the channel is identified by the case comms message ID (MID) in the case comms header:

Enumerator
DTS_CC_MID_MANAGEMENT 

DTS Management sub-channel, used by case firmware to communicate with DTS over case comms.

DTS_CC_MID_TUNNEL 

DTS Tunnel sub-channel, used by external host tunneling DTS to Earbuds through the case. This is equivalent to DTS over SPP direct to the Earbuds.
